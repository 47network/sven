# Verified Publisher Badges

Date: 2026-02-21

## Purpose

Verified badges identify publishers with a consistent track record of secure, maintainable skill contributions.

## Eligibility

- At least one accepted skill submission.
- No unresolved critical/high security findings in submitted skills.
- Maintainer identity confirmation (project-defined verification path).
- Agreement to security disclosure and update expectations.

## Badge Lifecycle

1. Publisher requests verification.
2. Maintainers review submission history and security posture.
3. Badge is granted, denied, or deferred with reasons.
4. Badge may be revoked if policy or security requirements are violated.

## Scope

- Badge indicates publisher verification status.
- Badge does not bypass runtime policy, approvals, or quarantine controls.
