## Actions
- create-machine: Define a new state machine with states and transitions
- send-event: Send an event to trigger a transition
- get-state: Get current machine state and context
- get-history: Retrieve full transition history
- terminate: Terminate a running state machine
- visualize: Generate state machine diagram
