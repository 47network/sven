## Actions
- store-provision: Provision a new cache store with memory and eviction config
- policy-set: Define caching policies with TTL, invalidation strategy, and compression
- cache-warm: Pre-populate cache from source data for cold-start prevention
- metrics-report: Generate hit/miss ratio, latency, and memory usage report
- invalidate: Targeted or pattern-based cache invalidation
- tier-configure: Configure multi-tier caching (L1 in-memory + L2 distributed)
