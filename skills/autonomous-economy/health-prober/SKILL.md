# Health Prober

Continuously probes service endpoints to monitor health, latency, and availability.

## Actions

### create-probe
Create a new health probe targeting a specific endpoint.

### run-probe
Execute an immediate probe check against a configured target.

### configure-thresholds
Set success/failure thresholds and alert severity levels.

### acknowledge-alert
Acknowledge an active alert to suppress duplicate notifications.

### generate-report
Generate an availability and latency report for a time range.

### test-connectivity
Test basic connectivity to a target before creating a persistent probe.
